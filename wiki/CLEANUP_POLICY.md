# Cleanup Policy

## Classification Rules

### Artifact

Artifact means generated runtime or research output.
Examples:

- `*.json`, `*.jsonl`, `*.csv`, `*.db`, `*.pkl`
- per-game deep reports generated by runtime flows
- logs and monitoring outputs
- postmortem outputs generated as research artifacts

Artifacts stay outside the wiki.

### Knowledge

Knowledge means durable repository guidance or decisions.
Examples:

- entrypoint definitions
- architecture boundaries
- cleanup history
- operational constraints
- governance and migration notes

Knowledge belongs in `wiki/` when it is current and canonical.
Historical knowledge remains in `docs/` or `docs/archive/`.

## Safety Gates

### Keep

Mark a file or directory as KEEP when any of the following is true:

- it is actively imported or executed
- it is the current runtime path
- it is required for reproducibility
- it is part of the historical record that should not be removed

### Migrate To Wiki

Mark a file as MIGRATE_TO_WIKI when:

- it contains current operational knowledge
- it is canonical guidance rather than raw output
- its content should become discoverable by agents and maintainers

Migration to wiki does not imply deleting the source file immediately.

### Backup And Archive

Mark a file as BACKUP_AND_ARCHIVE when:

- it is historical but still useful for traceability
- it should not remain in the active canonical path
- it may be needed for rollback or audit

### Delete Candidate

Mark a file as DELETE_CANDIDATE only when:

- it has no active imports or runtime path
- it has no remaining documentation dependency
- it has already been backed up if historical value exists
- its replacement path is already in place

Delete candidate does not mean safe to remove now.

## Required Before Any Deletion

- verify imports and call paths
- verify documentation references
- create a backup snapshot or archive location when historical value exists
- record the change in cleanup documentation

## Current Hard Constraints

- do not delete root `models/` or `strategy/` while `main.py` still imports them
- do not treat all of `data/wbc_backend/reports/` as deletable; classify file by file
- do not move or delete runtime snapshots and research artifacts without backup
