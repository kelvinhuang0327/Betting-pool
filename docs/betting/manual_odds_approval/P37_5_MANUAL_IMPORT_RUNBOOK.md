## Overview

This runbook walks through every step to:
1. Prepare a valid `odds_approval_record.json`
2. Prepare a valid `odds_2024_approved.csv`
3. Validate both files with the P37 gate
4. Confirm P37 is READY before proceeding to P38

No automation, scraping, or downloading is performed by this pipeline.
All steps require manual operator action.

## Step 1: Review Provider ToS

Use `P37_5_PROVIDER_TOS_REVIEW_TEMPLATE.md` to document your review.

Key requirements:
- Provider must allow `internal_research` use
- `commercial_use_allowed` must be `false`
- Provider may prohibit redistribution (that is acceptable)

Sources that are commonly research-friendly (not an endorsement):
- Publicly archived historical odds datasets with CC licenses
- Academic datasets purchased with explicit research license
- Self-collected manual odds from publicly accessible displays

---

## Step 2: Obtain the Odds Data File

**Method A — Purchased/licensed dataset**:
1. Purchase from provider under research license
2. Download to a local staging location (NOT inside the repo)
3. Move the relevant file to: `data/mlb_2024/manual_import/odds_2024_approved.csv`

**Method B — Manual entry**:
1. Use `data/mlb_2024/processed/p37_manual_odds_provisioning/odds_2024_approved_TEMPLATE.csv` as format reference
2. Manually enter odds for each game from a source with research license
3. Save as: `data/mlb_2024/manual_import/odds_2024_approved.csv`

**Important**: Never infer `p_market` from game outcomes.  Use closing odds only:
```
p_market = 1 / odds_decimal_home  (then vig-adjust if both sides available)
```

## Step 5: Validate Schema of Odds CSV

Quickly check the odds CSV has required columns:
```bash
head -1 data/mlb_2024/manual_import/odds_2024_approved.csv
```

Expected header:
```
game_id,game_date,home_team,away_team,p_market,odds_decimal,sportsbook,market_type,closing_timestamp,source_odds_ref,license_ref
```

Check no forbidden columns are present:
```bash
head -1 data/mlb_2024/manual_import/odds_2024_approved.csv | \
  grep -E "y_true|final_score|home_score|away_score|winner|outcome|result|run_diff|total_runs|game_result" \
  && echo "LEAKAGE_DETECTED" || echo "NO_LEAKAGE"
```

## Troubleshooting

| Error | Cause | Fix |
|-------|-------|-----|
| `P37_BLOCKED_APPROVAL_RECORD_MISSING` | No approval record file | Complete Steps 4–5 |
| `P37_BLOCKED_APPROVAL_RECORD_INVALID` | Missing fields or bad values | Fix fields per error message |
| `P37_BLOCKED_LICENSE_NOT_APPROVED` | `internal_research_allowed=false` or bad `allowed_use` | Fix provider/allowed_use |
| `P37_BLOCKED_MANUAL_ODDS_FILE_MISSING` | Approval OK but no odds CSV | Complete Steps 2–3 |
| `P37_BLOCKED_MANUAL_ODDS_SCHEMA_INVALID` | Missing columns or forbidden columns | Fix CSV schema |
| `P37_BLOCKED_RAW_ODDS_COMMIT_RISK` | Staged raw files detected | `git reset HEAD <file>` |
