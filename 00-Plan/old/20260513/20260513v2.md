### 8. 建議今天優先處理的方向

#### CTO 判斷

目前最大阻塞不是程式碼，而是 **資料授權與資料佈建**。
因此下一階段最值得優化的方向不是再寫更多 replay code，而是把「manual source gate」真正解鎖，或在 odds 等待期間並行推進 2024 OOF prediction rebuild feasibility。

目前 roadmap 與實際狀態的落差：

* 原先往 P38 odds artifact 前進的假設太樂觀。
* P37.5 已證明流程可控，但 **manual odds files 尚未存在**。
* 因此 P38 應降級為「P37 READY 後才能啟動」。
* 今日最實質可推進的工程方向應是：

  1. P0：執行 manual odds package readiness check + 若缺檔就輸出 operator action packet。
  2. P1：並行做 2024 OOF prediction rebuild feasibility，不等 odds。
  3. P2：建立 P38 preflight，僅在 P37 READY 後允許導入 odds artifact。

## Roadmap 重新排序：P0–P10

| Priority | Phase     | 最新建議方向                                                            | 狀態 / 理由                                  |
| -------- | --------- | ----------------------------------------------------------------- | ---------------------------------------- |
| **P0**   | **P37.6** | Manual Odds Provisioning Readiness Check / Operator Action Packet | 最高優先，因 P38 被 manual odds 缺檔阻擋            |
| **P1**   | **P38A**  | 2024 OOF Prediction Rebuild Feasibility Audit                     | 可並行，不依賴 odds；補另一半 joined input           |

| **P3**   | **P39**   | Build 2024 Joined Input Artifact                                  | 需要 odds + predictions 都 ready            |
| **P4**   | **P40**   | 2024 True-Date Replay Smoke                                       | 僅在 joined input ready 後                  |
| **P5**   | **P41**   | 2024 Full True-Date Replay Expansion                              | 類似 P27，但用 2024 joined input              |
| **P6**   | **P42**   | 2024 Stability / Sample Density Audit                             | 類似 P28/P29                               |
| **P7**   | **P43**   | Strategy Policy Re-evaluation on 2024                             | 需完整 2024 backfill evidence               |

這份排序與早期 CTO roadmap「P16 recommendation gate / risk hardening 優先」的精神一致：先補足可稽核資料與 gate，再談 recommendation / production。早期 roadmap 也明確要求 production_ready 保持 false、不得 live TSL、不得真實下注，並要求 recommendation / risk gate 先落地；目前 P37.5 延續了這個治理原則。
